## Git & GitHub Workflow

This project is connected to https://github.com/lior8755/claude-test.

Commit and push frequently throughout all work — after every meaningful unit of progress (new feature, bug fix, refactor, config change). Never batch up large amounts of work before committing. The goal is that the GitHub remote always reflects the latest stable state so work is never lost and any point can be reverted to.

- Write clean, descriptive commit messages that explain what changed and why
- Push to `origin main` after every commit
- Use `gh` CLI for any GitHub operations (repo is already configured)
